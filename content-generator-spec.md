# Content Generator Tab — Implementation Spec

## Overview

Add a new top-level page to the SEO Safety Intel Django dashboard for generating brand+category excerpts using OpenAI GPT-5.4 Mini. The page has three editable/output fields and calls the OpenAI API server-side.

## Page

- **Route:** `/content-generator/`
- **Nav label:** "Content Generator"
- **Accessible from:** top navigation bar alongside existing pages (Overview, Detail, Duplicates, Issues)

## UI Layout

Three fields stacked vertically or in a two-column + output layout:

### Field 1: System Prompt

- **Type:** `<textarea>`, editable
- **Rows:** ~10
- **Pre-filled default:**

```
For the following categories I need an excerpt based on the below information:

1. The excerpt should be in Turkish language.
2. The excerpt must highlight the main USP of a specific brand in connection to the respective category. Like Samsung Air conditioner should focus on the latest USP being pushed from Samsung in 2026 for Air conditioners. This can be technology, mechanical change, size change, or any other promotable USP.
3. The excerpt must not exceed 150 words.
4. The excerpt should not use repetitive sentence structure for different pages.
5. The excerpt should give an advertising feel and not by any chance feel like it is generated by AI.
6. The excerpt should follow proper Turkish semantics and grammatical structure.
7. This excerpt will be placed on the landing pages for each of the brand+category combination.
```

- User can modify this freely per session.

### Field 2: Input Context

- **Type:** `<textarea>`, editable
- **Rows:** ~10
- **Placeholder:** "Paste brand, category, product details here..."
- Contains the brand/category/product-level information that feeds the excerpt generation.

### Field 3: Output Excerpt

- **Type:** `<textarea>` or styled `<div>`, read-only
- **Rows:** ~8
- Shows the generated excerpt after the user clicks Generate.
- Includes a **Copy to Clipboard** button.

### Actions

- **Generate** button: triggers the API call. Disabled while request is in flight. Shows a loading spinner or "Generating..." state.
- **Clear** button (optional): resets input context and output.

## Backend

### View

- **GET** `/content-generator/` — renders the page with empty/default fields.
- **POST** `/content-generator/` — receives system prompt + input context, calls OpenAI, returns the generated excerpt.
- Can also be implemented as an AJAX endpoint (`POST /api/generate-excerpt/`) that returns JSON `{"excerpt": "..."}` for a smoother UX without full page reload.

### OpenAI Integration

- **SDK:** `openai` Python package (latest)
- **Model:** `gpt-5.4-mini`
- **API key:** read from environment variable `OPENAI_API_KEY`
- **Call structure:**

```python
from openai import OpenAI

client = OpenAI()  # reads OPENAI_API_KEY from env

response = client.chat.completions.create(
    model="gpt-5.4-mini",
    messages=[
        {"role": "system", "content": system_prompt},
        {"role": "user", "content": input_context},
    ],
    max_tokens=500,
    temperature=0.7,
)

excerpt = response.choices[0].message.content
```

### Error Handling

- If `OPENAI_API_KEY` is not set: show a user-friendly message on the page ("OpenAI API key not configured").
- If OpenAI returns an error (rate limit, invalid key, etc.): show the error message in the output field.
- Timeout: set a reasonable timeout (30s) on the API call.

## API Key Configuration

The key is **not exposed in the UI**. It lives server-side only.

### Local Development

Set in terminal before running Django:

```bash
export OPENAI_API_KEY="sk-..."
cd dashboard
python manage.py runserver
```

Or use a `.env` file with `python-dotenv`:

```text
OPENAI_API_KEY=sk-...
```

### Kubernetes / Production

Add as a Secret and reference in the deployment:

```yaml
env:
  - name: OPENAI_API_KEY
    valueFrom:
      secretKeyRef:
        name: openai-secret
        key: api-key
```

### Django Settings

```python
import os
OPENAI_API_KEY = os.environ.get("OPENAI_API_KEY", "")
```

## Styling

- Match the existing SEO Safety Intel dark theme (CSS variables, card styles, fonts).
- Fields should use the same `.card` container and form styling as the existing filter/table UI.
- Output field should have a slightly different background to distinguish it from inputs.

## Not In Scope

- No persistence / MongoDB storage for generated excerpts.
- No batch generation (one excerpt at a time).
- No history of past generations.
- No user authentication beyond what the dashboard already has.
